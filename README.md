# 3D Конфигуратор помещения

Интерактивная 3D-визуализация комнаты с возможностью настройки размеров, выбора типа раскладки напольного покрытия и автоматическим расчётом материалов.

Демо: https://gevorg22.github.io/react-three-app/

## Как запустить

Для запуска нужен Node.js версии 18 и выше.

```bash
npm install
npm run dev
```

После этого открыть в браузере [http://localhost:5173](http://localhost:5173).

Для production-сборки:

```bash
npm run build
```

## Если браузер показывает «WebGL недоступен»

Такое случается, когда Chrome отключает аппаратное ускорение - обычно после нескольких сбоев GPU или на виртуальных машинах.

Чтобы исправить: откройте `chrome://settings/system`, включите «Использовать аппаратное ускорение (GPU)» и перезапустите браузер. Если это не помогло, откройте проект в Firefox или Safari — там WebGL работает по умолчанию без дополнительных настроек.

## Тесты

Тесты написаны на Vitest и покрывают чистые функции без зависимостей от браузерного окружения.

`src/utils/calculations.test.ts` проверяет функцию `calculate`: корректность площади пола, стен, периметра, расчёт количества плашек с запасом для прямой раскладки и ёлочки, расход краски, граничные случаи с нулевым запасом и нестандартными размерами.

`src/utils/floorLayout.test.ts` проверяет `generateStraight` и `generateHerringbone`: непустой результат, корректные типы координат, покрытие комнаты, смещение нечётных рядов для прямой раскладки, наличие обоих направлений плашек в ёлочке, поведение при gap = 0 и масштабирование при увеличении комнаты.

```bash
npm test
```

Для запуска в режиме наблюдения:

```bash
npm run test:watch
```

Для сбора покрытия:

```bash
npm run test:coverage
```

## Стек

React 19, TypeScript, Vite, React Three Fiber, Three.js, @react-three/drei.

## Что умеет

Через слайдеры в боковой панели задаются ширина, длина и высота комнаты, размеры одной плашки, ширина шва и процент запаса материала. Всё обновляется в реальном времени.

Пол строится двумя способами. Ёлочка - классический паркетный паттерн, где плашки чередуются под углом 90 градусов друг к другу и укладываются под 45 градусов к стенам комнаты. Прямая раскладка — стандартные ряды со смещением через ряд на половину длины плашки.

Оба варианта рендерятся через `InstancedMesh`, так что сцена остаётся производительной даже при большом количестве плашек.

Стены покрашены в выбранный цвет, по периметру стоит плинтус с правильными угловыми стыками. Освещение состоит из рассеянного света, направленного источника с тенями и точечного света под потолком для имитации люстры.

В панели расчётов выводятся: площадь пола и стен, погонаж плинтуса, количество литров краски, количество плашек без запаса и с учётом запаса.

## Структура проекта

```
src/
  types.ts                  общие TypeScript-интерфейсы
  utils/
    calculations.ts         расчёт краски, плинтуса и плашек
    floorLayout.ts          алгоритмы генерации раскладки
  components/
    Room.tsx                корневая 3D-сцена
    Floor.tsx               плашки пола через InstancedMesh
    Walls.tsx               стены и пол-подложка
    Baseboard.tsx           плинтус по периметру
    Lighting.tsx            освещение
    ControlPanel.tsx        боковая панель управления
  App.tsx                   корневой компонент и состояние параметров
```

## Допущения

Площадь стен считается без вычета проёмов под окна и двери, поскольку их размеры и расположение не заданы в условии.

Расход краски принят как 8 квадратных метров на литр при двух слоях нанесения. Это усреднённый показатель для матовых интерьерных красок.

При раскладке ёлочкой автоматически добавляется 5% сверх заданного запаса. Это компенсирует угловые подрезы, которых при диагональной укладке значительно больше, чем при прямой.

Размер плашки по умолчанию 600 на 100 миллиметров - это стандартная планка для штучного паркета и ламината.

Зазор между плашками учитывается в шаге раскладки при генерации позиций. Геометрия каждой плашки при этом остаётся неизменной, визуально зазор проявляется как просвет между мешами.

Камера ограничена сверху и снизу: нельзя уйти ниже уровня пола и подняться выше зенита.

Предупреждение `THREE.Clock deprecated` в консоли браузера исходит из внутреннего кода React Three Fiber и не относится к коду проекта. Это известное несоответствие между R3F 9.x и Three.js 0.174+.
